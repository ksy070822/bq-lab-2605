2단계 — GCP 프로젝트 만들기

구글 콘솔에서 내 자리 만들기

1

console.cloud.google.com 접속 → 구글 계정 로그인

2

처음이면 "무료로 시작하기" 클릭 → 결제 정보 등록

이름·주소·카드 번호 입력 (구글이 $1 테스트 결제 후 즉시 취소)
국내 카드 안 되면 → 구글 계정 설정에서 국가 "대한민국"으로 변경

3

상단 "프로젝트 선택" → "새 프로젝트"

4

이름 입력 후 "만들기" (예: my-life-data)

결제 계정 자동 연결됨 (방금 등록한 거)

5

만들어진 프로젝트 ID 확인

이름 아래 작게 표시 (예: my-life-data-392847)

6

(권장) 결제 → "예산 및 알림" → 월 ₩5,000 알림 설정

💡 잠깐, 결제 정보 등록이 처음이라 걱정되세요?

오늘 실습 규모(수MB)에서 실제 청구액은 ₩0이에요. 카드는 본인 인증용이에요.
BigQuery는 매월 10GB까지 무료 — 우리 실습은 전체 합쳐 1MB 안 돼요.

결제/카드 등록에서 막혔어요? — AI한테 물어보기

아래 텍스트를 복사해서 Claude 또는 Gemini에 붙여넣어!

안녕! 나 GCP 처음 써보는데 결제 계정 등록에서 막혔어. 상황은 이래: - 구글 계정은 있어 - console.cloud.google.com 들어가서 "무료로 시작하기" 눌렀어 - 근데 [카드가 거절됨 / 결제 페이지가 안 열림 / 국가 설정 못 바꾸겠음] ← 내 상황으로 바꿔서 적을게 단계별로 알려줘. 한국에서 신용·체크카드로 등록하려고. 쉬운 말로!

3단계 — 데이터셋 만들기

데이터 넣을 폴더 하나. AI가 알아서 만들어줘요

💡 데이터셋이 뭐예요?

구글드라이브의 폴더처럼 생각하면 돼요.
프로젝트 → 데이터셋(폴더) → 테이블(엑셀 시트) 구조예요.
오늘 우리는 my_life라는 폴더를 만들고 그 안에 나의 유튜브 / 음악감상 / 웹서핑 등 인터넷 활동기록을 넣을 거예요.

⭐ 메인 방법 — AI가 알아서 만들어줘요

터미널에서 클로드 코드(또는 제미나이)한테 부탁하면 끝. 콘솔 클릭 안 해도 돼요.

① AI가 내 GCP에 접근할 수 있게 인증하기 (한 번만)
1

터미널 열고 gcloud CLI 설치 확인

맥: brew install google-cloud-sdk · 윈도우: 설치 가이드

2

로그인 (브라우저 열림 → 구글 계정 인증)

gcloud auth login

3

AI가 쓸 인증 발급 (한 번 더 브라우저 열림)

gcloud auth application-default login

4

내 프로젝트로 기본 설정

gcloud config set project [내 프로젝트 ID]

② AI한테 부탁하기

아래 프롬프트 복사 → Claude Code(터미널) 또는 Gemini CLI에 붙여넣기

안녕! 내 GCP 프로젝트에 빅쿼리 데이터셋 하나 만들어줘. 조건: - 프로젝트 ID: [2단계에서 만든 프로젝트 ID] - 데이터셋 이름: my_life (언더바 주의) - 위치: asia-northeast3 (서울) - gcloud 인증은 이미 해뒀어 (application-default-credentials 있음) 직접 만들고 bq ls 로 생성 확인까지 해줘. 어떤 명령어 썼는지도 알려주면 좋아!

💬 AI가 bq mk --location=asia-northeast3 my_life 같은 명령어 한 줄 실행해줄 거예요.
결과로 Dataset 'PROJECT:my_life' successfully created. 나오면 성공!

🛠 직접 콘솔에서 만들고 싶으면 (선택)

콘솔 UI로 직접 만들어보고 싶다면 이렇게:

1

좌측 메뉴 "BigQuery" 클릭

2

내 프로젝트 옆 ⋮ → "데이터셋 만들기"

3

데이터셋 ID: my_life (언더바!)

4

위치: asia-northeast3 (서울)

5

"데이터셋 만들기" 클릭

막혔어요? — AI한테 더 자세히 물어보기
안녕, 나 빅쿼리 데이터셋 만들려는데 막혔어. 상황: - 프로젝트 ID: [2단계에서 만든 프로젝트 ID] - gcloud auth login도 했어 - 근데 [bq 명령어를 못 찾겠음 / 권한 오류 / 데이터셋 만들기 버튼이 안 보임] ← 내 상황으로 바꿀게 목표: my_life 데이터셋, asia-northeast3 위치 생성. 단계별 쉽게 알려줘!

5단계 — 빅쿼리에 파일 올리기

다운받은 CSV → BigQuery 테이블. AI가 알아서 올려줘요

⭐ 메인 방법 — AI가 파일 잡아서 올려줘요

① CSV 파일이 있는 폴더에서 Claude Code 시작
1

youtube_history.csv · music_history.csv · web_history.csv · shop_history.csv

맥: 폴더에서 우클릭 → "Claude Code로 열기"
또는 채팅에 이 폴더 안 파일들 좀 봐줘: /Users/내이름/Downloads/

② AI한테 부탁하기 (이거 한 번만 보내면 끝)

현재 폴더에서 Claude Code 또는 Gemini CLI 실행 후 붙여넣기

안녕! 지금 내가 있는 이 폴더 안에 4개의 CSV 파일이 있어: - youtube_history.csv (유튜브 시청기록) - music_history.csv (음악 재생기록) - web_history.csv (웹서핑 기록) - shop_history.csv (쇼핑 기록) 이 파일들을 내 빅쿼리에 올려줘. 폴더만 잡아주면 파일 찾는 거, 스키마 추측하는 거, 업로드까지 알아서 해줄 거지? 기본 설정: - 프로젝트 ID: [2단계에서 만든 프로젝트 ID] - 데이터셋: my_life (asia-northeast3, 이미 있음) - 테이블 이름은 파일명에서 .csv 빼고 그대로 - 헤더 행 1줄 스킵, 스키마 자동 감지 - 날짜·시각 컬럼은 TIMESTAMP로 올린 다음 각 테이블 건수 확인까지 한 번에 해주고, 결과 보여줘!

💬 AI가 알아서 bq load --autodetect --skip_leading_rows=1 my_life.youtube_history youtube_history.csv 같은 명령어 실행해줄 거예요.
끝나면 youtube_history: 2547 rows 같은 결과 보여줘요.

2단계 — Slack에 내 봇(앱) 만들기

슬랙 개발자 페이지에서 봇 한 마리를 만들어줄게요

1

api.slack.com/apps 접속

슬랙 계정으로 로그인하세요 (구글 계정 아님!)

2

우측 상단 "Create New App" 클릭

3

"From scratch" 선택

처음부터 직접 만들기예요

4

앱 이름 입력

예: 내데이터리포트 — 나중에 봇 이름으로 표시돼요

5

워크스페이스에서 KakaoMobility 선택 → "Create App"

여기서 막혔어요? — AI한테 물어보기
Slack API에서 봇 앱을 만들려고 하는데 막혔어요. api.slack.com/apps 에서 "Create New App" 누르고 진행 중인데 워크스페이스 선택 화면이나 로그인 문제가 생겼어요. 쉽게 설명해줄 수 있어요?
🔑

3단계 — 봇 권한 설정 & 토큰 발급받기

봇이 메시지 보낼 수 있도록 권한 주고, 토큰(열쇠) 받기

1

왼쪽 메뉴 "OAuth & Permissions" 클릭

2

"Bot Token Scopes" → "Add an OAuth Scope" 클릭

3

chat:write 검색해서 추가

채널에 메시지 보내는 권한이에요. 이것 하나면 충분해요

4

맨 위로 올려서 "Install to Workspace" 클릭 → 허용

5

나타나는 "Bot User OAuth Token" 복사 📋

xoxb-로 시작하는 긴 문자열이에요. 잘 저장해두세요!

토큰 발급에서 막혔어요? — AI한테 물어보기
Slack OAuth & Permissions에서 chat:write 권한 추가하고 Install to Workspace 눌렀는데 문제가 생겼어요. 어떻게 해야 하나요?

4단계 — 사내 승인 신청하기

카카오모빌리티 슬랙에 봇 설치하려면 필수예요

🔒 카카오모빌리티 슬랙에 봇을 설치하려면 보안팀 + 경영지원팀 승인이 필요해요.
오늘 실습 중에 신청만 해두고, 승인 오면 바로 사용할 수 있어요!
1

아래 버튼으로 아지트 신청 게시판 열기

무엇이든 지원해드립니다 채널이에요

2

게시글 작성: 앱 이름, 용도, 본인 연락처

예: "내 빅쿼리 데이터를 주간 자동 리포트로 받는 슬랙 봇 설치 요청드립니다"

3

승인 대기 ☕

보통 며칠~1주일 걸려요. 오늘은 신청만 하고 다음 단계 미리 준비해요!

📋 아지트 신청 게시판 열기 →
📢

5단계 — 봇을 내 채널에 초대하기

승인 완료 후 진행해요. 봇이 채널 멤버여야 메시지를 보낼 수 있어요

✅ 승인 완료 후 진행하세요! 승인 전엔 이 단계가 안 돼요
1

슬랙에서 리포트 받을 채널 열기

없으면 채널 새로 만들기 (예: #내-데이터-리포트)

2

메시지 창에 /invite @봇이름 입력 후 전송

봇이 채널 멤버가 되어야 메시지를 보낼 수 있어요

3

채널 ID 확인하기 (다음 단계에서 입력!)

채널 이름 우클릭 → "링크 복사" → URL 끝 부분 11자 (예: C0123ABCDEF)
또는 데스크톱 슬랙에서 채널 정보 → 맨 아래 채널 ID 복사

Apps Script 실행했더니 오류 났어요?

오류 메시지를 아래에 붙여넣으면 → 옆 탭의 현재 코드 + 오류를 AI한테 통째로 전달해서 수정된 코드를 받아올 수 있어요.

💡 AI한테서 받은 수정 코드는 → 옆 탭의 코드 박스에 직접 붙여넣어 사용하시거나, 아래에 붙여넣어 보관하세요.

📥 AI가 준 수정 코드 여기에 붙여놔두기 (선택)
🚀 ⑤ 지금 바로 슬랙으로 보내보기
1

script.google.com → 새 프로젝트 만들기

상단 "새 프로젝트" 클릭 → 이름은 아무거나 (예: 내데이터슬랙봇)

2

BigQuery 서비스 연결하기

왼쪽 패널 "서비스 +" 클릭 → BigQuery API 검색 → 추가

⚠️ 이걸 빠뜨리면 오류나요 — 꼭 추가해주세요

3

위 코드 전체 복사 후 붙여넣기

에디터의 기본 코드 전체 지우고(Ctrl+A → Delete) → 위 코드 붙여넣기(Ctrl+V)

4

함수 선택 → sendWeeklyReport → ▶ 실행

상단 드롭다운에서 sendWeeklyReport 선택 후 실행 버튼 클릭

5

권한 허용 팝업 → "고급" → "허용"

"Google이 확인하지 않은 앱" 경고가 떠요 → 고급 클릭 → "안전하지 않은 페이지로 이동" → 허용

6

🎉 슬랙 확인! 메시지가 왔나요?

선택한 채널에 리포트 메시지가 도착했으면 성공이에요!

⏰ ⑥ 이제 자동화 설정 (매주 자동으로!)
1

함수 선택 → createWeeklyTrigger → ▶ 실행

드롭다운에서 함수 바꿔서 실행하면 끝이에요

2

트리거 등록 확인

왼쪽 메뉴 ⏱ (트리거 아이콘) 클릭 → 목록에 sendWeeklyReport가 보이면 완료!
